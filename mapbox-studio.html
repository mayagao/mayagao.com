A Powerful New Mapmaking Tool Fit for Both Pros and Newbies
I started working on Mapbox Studio four months before its public launch. A main focus of the initial release was the Mapbox Studio style editor, a professional map design tool. As the first visual designer in the team, I was tasked to create a unified visual language system for Studio's editor and dashboard.
Style Editor
The style editor is essentially creating JSON files that consists of specific map styling rules. The main goal (and also challenge) of this project is finding ways to display abstract, technical specifications in a visual way that's closer to how designers think about problems. Basically a user can select a layer and then edit data sources and styling certain properties inside the editor.
The sketch below was an iteration I did in early August. From left to right it shows the layer list, layer source editor, data selection popover, filter popover, layer style editor, and property panel. The overall layout already went through several iterations before I joined the team, but the UI still lacks consistency. So I spent lots of time polishing various form elements such as the list views, tabs, and toggles. I worked closely with the Mapbox cartography team and conducted weekly user testings to discover new usability and verify design solutions.

Property Function
Property functions provides designers a way to adjust and edit values based on different zoom levels. It was difficult to communicate the concept as well as to help users visualize the effects of the function. The three main design challenges:
  1. It was tough to understand the exactly effects of a property function and know how how values change in between stops. To solve this we introduced a small ramp graph alongside each property value and a large detailed graph inside each value editing popover.
  2. Number properties (e.g. widths) use exponential functions which generate outputs by interpolating between stops while enum properties use categorical functions which return the exact stop value. Color properties are also interpolated. We designed three different styles to help users distinguish between these three types.
  3. It was easy to invalid property values. To solve this we added smart sorting (stops are always in ascending order with no duplicate) and other validation rules.

Here are some explorations I designed to solve these problems:
Function graph
The large detailed graph in property function popover helps users understand the difference between these three property function types. The small graphs alongside the property values are really detailed in this iteration. I ended simplified them a lot more.
Editable graph
People in user testings often expect to be able to edit the graphs. We did not go with this idea because it was hard to control accuracy. Instead I simplified the small graph to make them look less editable.
Position indicator and distinct styles
I added this diamond pointer to indicate values at current zoom level. Distinct small graph styles make it more glanceable which are non-functions and which are functions and the function types.
Dashboard
In addition to map design, Studio also serves as an online platform that connects all the different Mapbox products, including the data upload and processing pipeline and the developer APIs and SDKs. It includes tools and interfaces to help users upload and manage custom data, analyze API usage, and manage account. And since we are introducing a completely new interface with the launch, we added a small amount of onboarding information to help users orient different pages.
Home page
The goal of the home page is to help user find what they are looking for, no matter it's map design, data upload, or developer documentation.
Listing pages
I designed several versions of the style and tileset listing page. The tileset page also includes to data upload status popover.
Modals
Modals is a main UI component of both the Studio editor and the dashboard. Then challenge lies in explaining abstract concepts and displaying large amount of information in a very confined space.
New style modal
There were six styles users can pick as a base for their map design. The goal of this modal is to help making an informed decision about which style to pick based on different needs.
Layer composite modal
Composite means combining various data sources (for use cases like label collision detection). Here I tried to use illustration to explain a abstract concept.
What I have learned
Coming to the team at a later stage definitely makes the job more challenging since I have far less context about the product and the development history. Luckily, the team at Mapbox values deeply about open communication and they tried to document everything on Github. So a big part of what I do everyday at the beginning was just reading to better understand how certain decisions are made. I really benefited tremendously from the transparent culture there, which allows me to get context that would otherwise not be available. This experience really helped me understand the importance of documentation (and sometimes over-communication) for anything that requires collaboration. Later when I started work on other projects I would also try my best to communicate clearly on tickets why I am making certain design decisions.
